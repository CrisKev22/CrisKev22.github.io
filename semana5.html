Desarrollar aplicación FrontEnd con Framework JS

Definiciones


1. Framework JS: Un conjunto de herramientas y bibliotecas que facilitan el desarrollo de aplicaciones web al proporcionar estructuras y convenciones predefinidas.

2. Client Side Rendering (CSR): Técnica en la que el contenido de la página web se genera en el navegador del cliente en lugar de en el servidor.

3. Gestión de dependencias: Manejo de las bibliotecas y módulos externos necesarios para un proyecto, generalmente mediante gestores de paquetes como npm o yarn.

4. Herramientas de creación de proyectos: Utilidades que facilitan el inicio y configuración de nuevos proyectos, como create-react-app para React o Vue CLI para Vue.js.

5. Desarrollo basado en componentes: - JSX: Sintaxis que permite escribir estructuras HTML dentro de JavaScript, utilizada en frameworks como React. - PROPS: Propiedades que se pasan a los componentes para configurar su comportamiento y apariencia. - CHILDREN: Elementos hijos que se pueden incluir dentro de un componente en JSX.

6. Utilización de estilos en Framework JS: - Styles inline: Estilos aplicados directamente en el atributo `style` de los elementos HTML. - Style Sheets: Hojas de estilo externas que se enlazan a través de una etiqueta ``. - Styles Modules: Módulos de estilos que permiten el uso de clases locales en un componente. - Styles components: Librería para aplicar estilos en componentes de manera modular y reutilizable. - Framework CSS: Frameworks de diseño como Bootstrap o Tailwind que ofrecen estilos predefinidos y componentes.

Procedimientos


1. Desarrollo de una aplicación Front-End con un Framework JS:

  • Configurar y crear un nuevo proyecto utilizando herramientas como create-react-app o Vue CLI.
  • Organizar la aplicación en componentes reutilizables siguiendo la estructura del framework.
  • Implementar Client Side Rendering para generar el contenido en el navegador del cliente.

2. Gestión de dependencias y herramientas:

  • Instalar y gestionar dependencias usando gestores de paquetes como npm o yarn.
  • Utilizar herramientas de creación de proyectos para establecer la configuración inicial y estructural del proyecto.

3. Desarrollo basado en componentes:

  • Crear componentes utilizando JSX y manejar propiedades con PROPS y CHILDREN.
  • Aplicar estilos a los componentes usando inline styles, style sheets, style modules y libraries como styled-components.

4. Laboratorio 05A:

  • Desarrollar un componente que reciba datos a través de PROPS y los muestre en un componente hijo.
  • Implementar CHILDREN para pasar contenido y elementos adicionales a un componente.
