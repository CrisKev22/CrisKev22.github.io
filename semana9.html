Next JS – CSR / SSR - TYPESCRIPT

Definiciones


1. Next.js: Framework de React para la construcción de aplicaciones web estáticas y dinámicas, que soporta renderizado del lado del cliente (CSR) y del servidor (SSR).


2. CSR (Client-Side Rendering): Técnica en la que el contenido de la página web se genera en el navegador del cliente.


3. SSR (Server-Side Rendering): Técnica en la que el contenido de la página web se genera en el servidor antes de ser enviado al navegador del cliente.


4. Routing en Next.js: Manejo de la navegación y las diferentes vistas dentro de una aplicación Next.js utilizando el sistema de archivos y el componente `Link`.


5. Estilos en Next.js: Métodos para aplicar estilos a los componentes de Next.js, incluyendo CSS, SASS, CSS-in-JS y bibliotecas como styled-components.


6. Formularios en Next.js: Creación y manejo de formularios en aplicaciones Next.js, incluyendo la validación y el manejo del estado.


7. TypeScript: Superset de JavaScript que añade tipado estático opcional, permitiendo detectar errores en tiempo de desarrollo y mejorar la mantenibilidad del código.


8. De JavaScript a TypeScript: Proceso de migrar un proyecto de JavaScript a TypeScript, añadiendo tipos y ajustando el código para aprovechar las ventajas de TypeScript.

Procedimientos


1. Creación de proyectos en Next.js:

  • Configurar y crear un nuevo proyecto utilizando Create Next App.
  • Configurar el routing y la navegación entre diferentes páginas del proyecto.
  • Implementar estilos utilizando diversas técnicas y bibliotecas de estilos.
  • Crear y manejar formularios dentro de la aplicación Next.js.

2. Implementación de CSR y SSR en Next.js:

  • Configurar el proyecto para usar renderizado del lado del cliente (CSR) para mejorar la interactividad.
  • Configurar el proyecto para usar renderizado del lado del servidor (SSR) para mejorar la performance y el SEO.

3. Uso de TypeScript en proyectos Next.js:

  • Configurar un proyecto Next.js para usar TypeScript.
  • Convertir componentes y funciones de JavaScript a TypeScript, añadiendo tipos y ajustando el código.
  • Utilizar las características avanzadas de TypeScript para mejorar la calidad y mantenibilidad del código.

4. Laboratorio 10:

  • Desarrollar un caso práctico de consumo de una API en un proyecto Next.js utilizando TypeScript.
  • Implementar formularios y manejar el estado y la validación utilizando TypeScript.
