Desarrollar aplicaciones con Eventos, Condicionales y Bucles con Framework JS

Definiciones


1. Eventos en Framework JS: Mecanismos para manejar interacciones del usuario con la aplicación, como clics, cambios de formulario y movimientos del ratón.

2. Renderizado condicional: Técnica para mostrar o esconder componentes y elementos en función de ciertas condiciones o estados.

3. Renderizado iterativo: Método para generar listas y conjuntos de elementos mediante bucles, a menudo utilizando la función `map` en frameworks como React.

4. Formularios: Elementos HTML que permiten la entrada de datos del usuario, manejados en frameworks JS mediante estados y controladores de eventos.

5. Routing: Gestión de la navegación y las diferentes vistas dentro de una aplicación web utilizando bibliotecas de routing, como React Router.

6. Consumo de APIs: Proceso de obtener datos de servicios externos a través de interfaces de programación de aplicaciones (APIs).

7. Promesas: Objetos que representan la eventual finalización (o falla) de una operación asíncrona y su valor resultante.

8. Async await: Sintaxis para trabajar con promesas de manera más clara y concisa en funciones asíncronas.

9. Librería Axios: Biblioteca para hacer solicitudes HTTP desde el navegador y Node.js, facilitando el consumo de APIs.

Procedimientos


1. Manejo de eventos, condicionales y bucles en Framework JS:

  • Implementar manejadores de eventos para capturar interacciones del usuario y actualizar el estado de la aplicación.
  • Utilizar renderizado condicional para mostrar u ocultar componentes en función de condiciones específicas.
  • Aplicar renderizado iterativo para generar listas de elementos a partir de datos dinámicos.

2. Manejo de formularios y routing:

  • Crear y manejar formularios dentro de la aplicación, controlando su estado y validación.
  • Implementar rutas y navegación dentro de la aplicación utilizando bibliotecas de routing.

3. Consumo de APIs y manejo de datos:

  • Realizar solicitudes a APIs externas para obtener datos utilizando promesas y la sintaxis async await.
  • Utilizar la librería Axios para simplificar las solicitudes HTTP y manejar las respuestas de las APIs.

4. Laboratorio 06A:

  • Desarrollar una aplicación que maneje eventos y realice renderizado condicional e iterativo basado en datos obtenidos de una API.
  • Utilizar Axios para obtener los datos necesarios desde una API y mostrarlos en la interfaz de usuario.
