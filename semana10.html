Next JS – CSR / SSR - Static Generate – Static Regeneratión

Definiciones


1. Next.js: Framework de React para la construcción de aplicaciones web estáticas y dinámicas, que soporta renderizado del lado del cliente (CSR) y del servidor (SSR).


2. CSR (Client-Side Rendering): Técnica en la que el contenido de la página web se genera en el navegador del cliente.


3. SSR (Server-Side Rendering): Técnica en la que el contenido de la página web se genera en el servidor antes de ser enviado al navegador del cliente.


4. Static Generation (SG): Proceso de generar páginas HTML estáticas en el momento de la construcción del sitio, que se sirve en cada solicitud.


5. Incremental Static Regeneration (ISR): Técnica que permite actualizar el contenido estático después de la construcción inicial del sitio, generando nuevas páginas de manera incremental.


6. Incremental Static Generation (ISG): Técnica similar a ISR que permite la generación y actualización incremental de páginas estáticas en el momento de la solicitud.


7. Routing en Next.js: Manejo de la navegación y las diferentes vistas dentro de una aplicación Next.js utilizando el sistema de archivos y el componente `Link`.


8. Estilos en Next.js: Métodos para aplicar estilos a los componentes de Next.js, incluyendo CSS, SASS, CSS-in-JS y bibliotecas como styled-components.


9. Formularios en Next.js: Creación y manejo de formularios en aplicaciones Next.js, incluyendo la validación y el manejo del estado.


10. TypeScript: Superset de JavaScript que añade tipado estático opcional, permitiendo detectar errores en tiempo de desarrollo y mejorar la mantenibilidad del código.

Procedimientos


1. Creación de proyectos en Next.js:

  • Configurar y crear un nuevo proyecto utilizando Create Next App.
  • Configurar el routing y la navegación entre diferentes páginas del proyecto.
  • Implementar estilos utilizando diversas técnicas y bibliotecas de estilos.
  • Crear y manejar formularios dentro de la aplicación Next.js.

2. Implementación de CSR y SSR en Next.js:

  • Configurar el proyecto para usar renderizado del lado del cliente (CSR) para mejorar la interactividad.
  • Configurar el proyecto para usar renderizado del lado del servidor (SSR) para mejorar la performance y el SEO.

3. Static Generation y técnicas incrementales en Next.js:

  • Implementar Static Generation (SG) para generar páginas HTML estáticas en el momento de la construcción.
  • Utilizar Incremental Static Regeneration (ISR) para actualizar el contenido estático de manera incremental después de la construcción inicial.
  • Aplicar Incremental Static Generation (ISG) para generar y actualizar páginas estáticas en el momento de la solicitud.

4. Uso de TypeScript en proyectos Next.js:

  • Configurar un proyecto Next.js para usar TypeScript.
  • Convertir componentes y funciones de JavaScript a TypeScript, añadiendo tipos y ajustando el código.
  • Utilizar las características avanzadas de TypeScript para mejorar la calidad y mantenibilidad del código.

5. Laboratorio 10:

  • Desarrollar un caso práctico de consumo de una API en un proyecto Next.js utilizando TypeScript.
  • Implementar formularios y manejar el estado y la validación utilizando TypeScript.
